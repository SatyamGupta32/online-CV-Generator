# Week 6 Project Reflection - Online CV Generator

## 1) Project Summary

This project involved building and deploying a full stack Online CV Generator using React on the frontend and Node.js/Express with MongoDB on the backend. The final stage focused on production deployment, maintenance planning, and evaluating the complete development journey.

## 2) What Went Well

- Full stack architecture was successfully implemented end-to-end.
- Core user flow (authentication + resume management) was completed.
- Frontend and backend were both deployed on Render and made publicly accessible.
- Test and debugging documentation was prepared during development.
- Overall project goals were achieved within the required scope.

## 3) Challenges Faced

### Deployment and Environment Issues
- Ensuring the frontend pointed to the correct production backend URL.
- Managing environment variables safely across local and production environments.
- Handling backend startup/build issues during cloud deployment.

### Integration and Debugging
- Fixing API communication errors between client and server.
- Resolving authentication issues related to JWT handling and protected routes.
- Diagnosing database connectivity and request/response edge cases.

### Process Challenges
- Coordinating backend, frontend, and deployment changes together.
- Maintaining documentation quality while continuing development and testing.

## 4) How Challenges Were Solved

- Used structured debugging with logs and targeted test runs.
- Standardized environment variable setup and separated local/production config.
- Re-tested complete user flows after each deployment change.
- Maintained reports to capture issues, fixes, and optimization decisions.

## 5) Key Skills and Learning Outcomes

- Better understanding of full stack request lifecycle.
- Practical experience in deploying production-ready applications.
- Improved debugging approach using logs, tests, and incremental fixes.
- Stronger knowledge of authentication/security basics in web apps.
- Improved confidence in handling real deployment failures under constraints.

## 6) Maintenance Mindset Gained

This project reinforced that development does not end at feature completion. Deployment monitoring, bug triage, security updates, and documentation are essential to keep a full stack application reliable over time.

## 7) What I Would Improve in Future Projects

- Add CI/CD pipeline for automated tests before deployment.
- Improve UI/UX polish and responsive behavior across devices.
- Add advanced resume templates and customization options.
- Add richer telemetry/monitoring and centralized error tracking.
- Increase automated test coverage for critical backend endpoints and UI flows.

## 8) Personal Self-Improvement Plan

- Practice designing deployable architecture earlier in the project.
- Write tests in parallel with features (not only near deadlines).
- Improve release checklist discipline before production updates.
- Continue improving secure coding practices and dependency hygiene.

## 9) Final Reflection

This project provided realistic hands-on experience with the full software lifecycle: design, implementation, debugging, testing, deployment, and maintenance planning. The journey included technical setbacks, but those setbacks significantly improved practical engineering skills and readiness for real-world development work.
